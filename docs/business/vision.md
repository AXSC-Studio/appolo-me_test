## プロダクト概要

**「出張中に今リアルで会うべき人をAIが見つけ、アポイントまでつなぐサービス」**

- カテゴリ: 出張中の商談創出版・アポイント獲得補助
- ターゲット: 30代ブリージャー（BIZ × レジャー）層

## UVP

> 「日程を埋めるのではなく、成果を埋める」

出張目的・滞在予定・会いたい相手の条件を入力すると、AIが候補者を3名以上抽出し、優先順位・推薦理由・次アクションを提示する診断型MVP。

## ターゲットペルソナ

- 年齢層: 30代前半 / 性別: 男性
- 職業: デジタルノマド / 年収: 700〜1,000万円
- 特徴: Work as Life志向。旅先でも自然に仕事を組み込む

## 主要指標（UVP証明）

**重要人物面会実現率** = 実際に面会できた重要人物数 ÷ 面会対象として設定した重要人物数 × 100

## E2E 理想顧客体験

### ホスト（アポイントを獲得したい人）
1. サイトにアクセスする
2. 出張目的・滞在予定・会いたい相手の条件を入力する
3. AIが抽出した候補者ランキングを確認する
4. 優先順位・推薦理由・次アクションを確認する
5. 「次に会いたい相手」を選択する
6. アポイント依頼文または次アクションを確認する

### ゲスト（アポイントを入れたい人）
1. ホストから共有された案内URLまたは依頼内容を確認する
2. ホストの出張目的・会いたい理由を確認する
3. 自分に関係がある依頼か判断する
4. 日程候補または関心有無を選択する
5. アポイント可否を回答する

## v0 受け入れ条件（V字モデル対応）

### ❸' 単体テスト対象コンポーネント
1. 候補者DBコンポーネント: 名前・属性・接点・相性・想定メリット保持、3名以上表示
2. 出張情報入力コンポーネント: 出張目的・滞在予定・条件受取、未入力時不足判定
3. マッチング条件コンポーネント: 目的と候補者属性照合、優先順位評価軸保持
4. 推薦ロジックコンポーネント: スコア・順位・推薦理由を一貫して生成、入力変更時再計算
5. ランキング表示コンポーネント: 候補者を優先順位付きで表示、推薦理由と次アクションを同画面
6. 選択状態コンポーネント: 1名選択でき、選択結果が画面上で明確に保持
7. エラー/空状態コンポーネント: 入力不足・候補なし・処理失敗時に次の行動を提示

### ❷' 結合テスト
入力 → 候補抽出 → 優先順位表示 → 推薦理由表示 → 次に会う相手の選択まで、1つの主要導線として破綻なくつながること

### ❶' 受け入れテスト（5項目）
1. 出張目的を入力できる
2. 候補者が3名以上表示される
3. 優先順位が明確に見える
4. 各候補の推薦理由が分かる
5. 顧客が次に会う相手を選べる
